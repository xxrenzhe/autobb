## 1. 设计概览

### 1.1 设计目标

**用户体验目标**：
1. **简单直观**：非专业用户3分钟内学会创建广告
2. **高效流畅**：最小化操作步骤，每个流程≤3步
3. **即时反馈**：所有操作立即显示结果或进度
4. **可控透明**：AI辅助但用户保持最终控制权

**视觉设计目标**：
1. **专业可信**：企业级SaaS产品的视觉质量
2. **现代简洁**：遵循Material Design 3或Shadcn/ui设计系统
3. **一致性**：统一的颜色、字体、组件风格
4. **可访问性**：符合WCAG 2.1 AA标准

### 1.2 设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **Progressive Disclosure** | 渐进式展示信息，避免一次性呈现过多内容 | 高级选项默认折叠，点击"更多选项"展开 |
| **Intelligent Defaults** | 提供最佳实践的默认值 | 预算默认100/天，关键词匹配类型自动分类 |
| **Immediate Feedback** | 操作后立即显示结果 | 点击"生成创意"立即显示进度条 |
| **Error Prevention** | 防错设计优于错误提示 | 表单实时验证，禁用无效操作的按钮 |
| **User Control** | 用户可撤销、编辑AI生成的内容 | 提供"重新生成"、"编辑"、"恢复历史版本" |

**Offer卡片设计**：
- 卡片布局，3列网格
- 每张卡片显示：
  - Offer名称（标题）
  - 品牌名称
  - 推广国家（新增）
  - 关键词数量
  - 状态badge
  - 已创建广告数量
- 主要操作：
  - 大按钮"一键上广告"（蓝色填充）
  - 新增按钮"📊 投放分析"（紫色边框）
  - 新增按钮"💰 调整CPC"（橙色边框）
- 次要操作：编辑、删除（灰色边框）

**交互设计**：
1. **点击卡片** → 进入Offer详情页
2. **点击"一键上广告"** → 打开配置弹窗
3. **点击"📊 投放分析"** → 打开投放评分弹窗（见下方）
4. **点击"💰 调整CPC"** → 打开CPC调整弹窗
5. **点击"编辑"** → 进入编辑表单
6. **点击"..."菜单** → 显示更多操作（复制、归档、删除）

**新增字段说明**：

1. **推广链接 (promotionUrl)**：
   - 必填字段
   - 格式验证：必须为有效URL
   - 自动赋值给landingPageUrl
   - 用途：联盟追踪链接或带参数的推广链接

2. **推广国家 (targetCountry)**：
   - 必填字段
   - 下拉选择：US, UK, GE, FR, ES, IT, JP, CN等
   - 选择后自动推导targetLanguage
   - 影响AI生成的语言和广告定位

   - 如需多语言推广，需为每种语言创建独立的Offer

3. **推广语言 (targetLanguage)**：
   - 只读字段（自动推导）
   - 根据targetCountry自动设置
   - 映射关系：US/UK/CA/AU→English, GE→German, FR→French等
   - 显示为禁用状态，带🔒图标
   - 字段下方提示：💡 根据推广国家自动设置，如需多语言请创建多个Offer

4. **店铺/详情页 URL (productPageUrl)**：
   - 必填字段
   - 格式验证：必须为有效URL
   - 用于AI数据抓取和分析
   - 与promotionUrl不同：此URL用于读取商品信息，不带追踪参数

5. **Landing Page URL (landingPageUrl)**：
   - 只读字段（自动生成）
   - 自动等于promotionUrl
   - 显示为禁用状态，带🔒图标
   - 确保广告点击后访问带追踪的推广链接

- 错误处理：
  - API配额不足：显示"今日查询次数已用完，明天再试"
  - 无效国家：显示"该国家暂不支持关键词查询"
  - 关键词无效：显示"关键词'{keyword}'无搜索数据"

**智能提示**：
- 每个字段下方提供最佳实践建议
- 推广链接和店铺URL自动验证可访问性
- 关键词数量提示（建议5-10个）
- 国家选择后提示语言已自动设置

**保存选项**：
1. **保存并继续** → 保存后跳转到"一键上广告"流程
2. **保存为草稿** → 保存后返回Offer列表

**热更新实现机制**：
```typescript
// 配置更新流程
async function updateConfig(key: string, value: any) {
  // 1. 验证配置值
  validateConfigValue(key, value);

  // 2. 更新IndexedDB
  await db.configs.put({ key, value, updatedAt: new Date() });

  // 3. 触发全局配置更新事件
  window.dispatchEvent(new CustomEvent('config-updated', {
    detail: { key, value }
  }));

  // 4. 显示成功提示
  showToast('配置已更新并立即生效', 'success');
}

// 各模块监听配置更新
window.addEventListener('config-updated', (event) => {
  const { key, value } = event.detail;

  switch(key) {
    case 'gemini_api_key':
      // 更新Gemini客户端
      updateGeminiClient(value);
      break;
    case 'sync_frequency':
      // 重新调度同步任务
      rescheduleSyncTask(value);
      break;
    // ... 其他配置项
  }
});
```

- Large: 高度48px，用于CTA
- Medium: 高度40px，默认

**输入框（Input）**：
- 高度: 40px
- 边框: 1px solid #E5E7EB
- Focus: 蓝色边框 + 阴影
- 错误: 红色边框
- 禁用: 灰色背景

- 与Input样式一致
- 下拉箭头在右侧

**Badge**：
- 状态标识
- 颜色映射：
  - Active: 绿色

  - Error: 红色

### 4.4 反馈组件（Feedback）

**进度条（Progress）**：
- 高度: 8px
- 蓝色填充部分
- 显示百分比

**Toast通知**：
- 右上角弹出
- 自动消失（3秒）
- 类型: Success, Warning, Error, Info

**Modal弹窗**：
- 遮罩层半透明黑色
- 内容居中
- 最大宽度800px

## 5. 用户流程设计

### 5.1 首次使用流程

```
1. 用户访问AutoAds
   ↓
2. Google OAuth登录
   ↓
3. 连接Google Ads账号
   - 选择账号（如果有多个）
   - 授权API访问
   ↓
4. 进入Dashboard（首次为空）
   ↓
5. 引导创建第一个Offer
   - 显示"开始创建您的第一个广告"引导卡片
   - 点击"创建Offer" → 打开表单
   ↓
6. 填写Offer信息
   - 实时提示最佳实践
   - 保存Offer
   ↓
7. 引导"一键上广告"
   - 自动弹出配置弹窗
   - 选择变体数量
   ↓
8. AI生成创意
   - 观看生成过程
   - 查看质量评分
   - 确认创建
   ↓
9. Campaign创建成功
   - 查看成功页面
   - 完成后续步骤checklist
   ↓
10. 返回Dashboard
    - 等待数据同步（24小时后有数据）
```

### 5.2 日常使用流程

```
1. 用户打开AutoAds
   ↓
2. 自动触发数据同步
   - 显示"正在同步数据..."
   - 完成后显示Dashboard
   ↓
3. 用户查看Dashboard
   - 查看KPI卡片
   - 查看趋势图表
   - 查看智能洞察
   ↓
4. 用户根据洞察采取行动
   Option A: 创建新Offer
   Option B: 查看Campaign详情
   Option C: 应用优化建议
   Option D: 导出数据备份
```

### 5.3 A/B测试分析流程

```
1. 用户创建3个变体
   - 品牌导向
   - 功能导向
   - 价值导向
   ↓
2. 等待7-14天积累数据
   ↓
3. 在Dashboard查看对比
   - Campaign列表显示3个变体的数据
   - 对比CTR、CVR、CPA
   ↓
4. 识别获胜变体
   - 系统自动标识表现最佳的变体
   - 显示建议："品牌导向变体表现最佳，建议暂停其他变体并增加预算"
   ↓
5. 执行优化
   - 暂停表现差的变体
   - 增加获胜变体的预算
   - 或在Google Ads控制台操作
```
